Pace Design System

Tokens and components for building consistent, on-brand interfaces across the Pace product. Built dark-first, with one accent color used deliberately.

Colors

A near-black neutral scale with a single warm accent. Orange is reserved for actions and live/active states — it should never appear as decoration.

✓ Do

Use accent only on the single most important action in a view — a primary button, an active status dot, a focused input ring.

✕ Don't

Don't use accent for large fills, multiple competing buttons, or decorative backgrounds — it loses meaning as a signal.

Buttons

Four variants, three sizes, four states each. Primary is reserved for one action per view.

Variants (medium size)

Sizes

States

Primary — default / hover / disabled
Secondary — default / hover / disabled
✓ Do

Pair one primary button with at most one secondary/ghost button per view. Use btn-danger only for destructive, irreversible actions.

✕ Don't

Don't place two primary buttons side by side — it forces the user to guess which action actually matters.

Form elements

Inputs, selects, checkboxes, radios, and toggles. All states share the same focus ring treatment for consistency.

✓ Do

Always pair an error state with a specific, actionable message below the field — never just a red border alone.

✕ Don't

Don't rely on color alone to indicate state — disabled fields also drop opacity, error fields also show message text.
